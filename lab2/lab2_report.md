## 1. Описание интеграции

### Какие источники данных выбраны

Для бота «Погодный Советник» подключены **два источника данных**:

1. **Open-Meteo API** — получение актуальных погодных данных и прогнозов
2. **SQLite база данных** — хранение избранных городов пользователей

### Почему именно эти источники

- **Open-Meteo API** — полностью бесплатный, не требует API-ключей и регистрации. Предоставляет точные данные по всему миру: температуру, ветер, влажность, осадки, UV-индекс, время рассвета/заката. Поддерживает прогнозы до 16 дней.
- **SQLite** — встроена в Python, не требует установки сервера. Идеально подходит для хранения небольших объёмов пользовательских данных. Файл базы данных создаётся автоматически при запуске бота.

### Структура данных

**Open-Meteo API (погода):**

Запрос: `GET https://api.open-meteo.com/v1/forecast`

Параметры:
- `latitude`, `longitude` — координаты города
- `current` — текущие показатели (температура, ветер, влажность, осадки, код погоды)
- `daily` — прогноз по дням (мин/макс температура, осадки, ветер, UV-индекс, восход/закат)

**Open-Meteo Geocoding API:**

Запрос: `GET https://geocoding-api.open-meteo.com/v1/search`

Переводит название города → координаты (latitude, longitude). Поддерживает русский язык.

**SQLite (избранные города):**

```sql
CREATE TABLE favorite_cities (
    user_id   INTEGER,   -- ID пользователя Telegram
    city_name TEXT,       -- название города
    latitude  REAL,       -- широта
    longitude REAL,       -- долгота
    PRIMARY KEY (user_id, city_name)
);
```

## 3. Реализация

### Как работает интеграция с Open-Meteo API

Бот использует две конечные точки Open-Meteo:

1. **Geocoding** — пользователь пишет «Москва», бот получает координаты (55.75, 37.62)
2. **Forecast** — по координатам бот получает текущую погоду и прогноз

Ключевые фрагменты кода:

```python
def geocode(city_name: str) -> dict | None:
    resp = requests.get(
        GEOCODING_URL,
        params={"name": city_name, "count": 1, "language": "ru", "format": "json"},
        timeout=10,
    )
    data = resp.json()
    results = data.get("results")
    if not results:
        return None
    r = results[0]
    return {
        "name": r.get("name", city_name),
        "country": r.get("country", ""),
        "lat": r["latitude"],
        "lon": r["longitude"],
    }
```

```python
def fetch_weather(lat: float, lon: float) -> dict:
    resp = requests.get(

            "timezone": "auto",
            "forecast_days": 3,
        },

```

### Как работает SQLite

При запуске бот создаёт файл `favorites.db` и таблицу `favorite_cities`. Каждый пользователь может сохранять города (привязка через `user_id` из Telegram). При просмотре избранных генерируются inline-кнопки: `🌤` для текущей погоды и `📅` для прогноза.

```python
def add_favorite(user_id: int, city: str, lat: float, lon: float):
    conn = sqlite3.connect("favorites.db")
    cur = conn.cursor()
    cur.execute(
        "INSERT OR REPLACE INTO favorite_cities VALUES (?, ?, ?, ?)",
        (user_id, city, lat, lon),
    )
    conn.commit()
    conn.close()
```

## 5. Трудности и решения

| Проблема | Решение |
|---|---|
| Open-Meteo Geocoding иногда возвращает несколько результатов для неточных запросов | Ограничил результат до 1 (`count=1`) и добавил вывод страны для уточнения |
| Inline-кнопки имеют ограничение на размер callback_data (64 байта) | Передаю в callback данные в формате `action\|city\|lat\|lon` — укладывается в лимит |
| SQLite не потокобезопасен при одновременных запросах | Для текущей нагрузки (один пользователь за раз) это некритично; для масштабирования можно перейти на PostgreSQL |
| API может не ответить (таймаут, ошибка сети) | Добавлен `timeout=10` и `try/except` с информативным сообщением пользователю |
