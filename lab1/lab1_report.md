# Лабораторная работа №1

## «Создание первого Telegram-бота без программирования»

---

## 1. Описание задачи

### Какую проблему решает бот

«Погодный Советник» — Telegram-бот, который не просто показывает погоду в любом городе мира, но и выполняет роль персонального помощника: подсказывает **что надеть** и **чем заняться** в зависимости от текущих погодных условий.

Бот решает повседневную проблему: каждое утро мы тратим время, чтобы понять, нужен ли зонт, какую куртку надеть и стоит ли планировать прогулку. Бот собирает все эти решения в одном сообщении.

### Почему выбрана эта задача

- Погодные данные доступны бесплатно через Open-Meteo API (без ключей и регистрации)
- Бот полезен для любого пользователя — легко найти тестеров для будущих лабораторных
- Тема позволяет добавить «умную» логику (рекомендации по одежде и активностям), что делает бота интереснее обычного прогноза
- Возможность расширения: избранные города, сравнение, прогнозы

## 2. Промпт для LLM

### Исходный промпт

```
Создай Telegram-бота на Python с использованием библиотеки python-telegram-bot.

Функционал бота «Погодный Советник»:
- Показывает текущую погоду по названию города (температура, ветер, влажность, осадки)
- Даёт рекомендации что надеть в зависимости от температуры, осадков и ветра
- Предлагает чем заняться в зависимости от погоды (пробежка, кино, кафе, пикник и т.д.)
- Показывает прогноз на 3 дня вперёд
- Позволяет сохранять избранные города и быстро получать по ним погоду
- Имеет удобное меню с кнопками внизу экрана

Требования:
- Использовать Open-Meteo API (бесплатный, без ключей)
- Использовать Open-Meteo Geocoding API для поиска координат города
- Хранить избранные города в SQLite
- Все описания погоды на русском языке с эмодзи
- Обработка ошибок (город не найден, API не отвечает)
- Токен бота хранить в .env файле

Создай:
1. Файл bot.py с кодом бота
2. Файл requirements.txt с зависимостями
3. Файл .env.example для примера конфигурации
4. Файл README.md с инструкцией по запуску
```

### Итерации и улучшения

В процессе работы промпт уточнялся:
- Добавлен запрос на WMO-коды погоды (стандартные коды Open-Meteo) с русскими описаниями
- Уточнена логика рекомендаций по одежде — добавлены пороги для ветра и осадков
- Добавлена обработка inline-кнопок для избранных городов

### Финальный промпт

Финальный промпт совпадает с исходным, так как основная итерация происходила через точечные доработки кода с помощью LLM.

## 5. Трудности и решения

| Проблема | Решение |
|---|---|
| Open-Meteo возвращает коды погоды (WMO), а не текстовые описания | Создан словарь `WMO_DESCRIPTIONS` с русскими описаниями и эмодзи для каждого кода |
| Для API нужны координаты, а пользователь вводит название города | Подключён Open-Meteo Geocoding API, который переводит название в координаты |
| Токен бота может случайно попасть в git | Использован `.env` файл + `.gitignore`, токен загружается через `python-dotenv` |
| Сложно протестировать все погодные условия | Созданы отдельные функции `clothing_advice` и `activity_advice` с чёткими пороговыми значениями |
